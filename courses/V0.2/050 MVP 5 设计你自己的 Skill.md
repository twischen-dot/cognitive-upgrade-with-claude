## 这个课程的设计哲学

MVP 1 你体验了压缩。MVP 2 你走了消化管道。MVP 3 你用三轮迭代压缩了一本书。MVP 4 你用升维公式破解了自己的困境。

四次你都是工具的**使用者**。

你用过 xray-book、explain-concept、rank、升维公式——但你从没问过：这些工具本身是怎么设计出来的？为什么它们管用？如果你自己有一个反复遇到的认知问题，能不能设计一个工具来解决它？

MVP 5 的核心体验：先解剖一个已有的 Skill，看清它的骨架和基因，然后用 Ω = A + M · V 公式从零设计一个属于你的 Skill。

加上两道新的验证门（创造者检验 + 递归检验），完成从消费者到创造者的跃迁。

做一次就知道了。

## 一个故事

请阅读以下故事：

---

> **老王的手艺**
>
> 老王做了三十年木工。他做的榫卯家具，不用一颗钉子，严丝合缝，能传三代。方圆百里的人都来找他打家具。
>
> 老王收过很多徒弟。他教得很认真：手把手带，一个动作一个动作做示范。但奇怪的是，三十年来，没有一个徒弟能独立做出他水准的家具。
>
> 有人问他为什么。老王想了半天，说："我也不知道。就是觉得他们差点什么。可能是手感吧。"
>
> 有一天，一个年轻人来拜师。他和别的徒弟不同——他不急着学怎么做，而是问了一个奇怪的问题：
>
> "王师傅，您能不能不做家具，只是把您的手在做每一步时注意什么、判断什么写出来？"
>
> 老王觉得这个要求很古怪，但试了试。他发现：他真的写不出来。不是不愿意，而是他自己也不知道。他的手知道——哪里该用力、哪里该轻、木头的纹理是顺还是逆，他一摸就知道。但他说不出来。
>
> 年轻人没有放弃。他用了一个笨办法：每次看老王做一步，他就试着做同样的动作，然后让老王看哪里不对。老王虽然说不出"怎么做"，但一看就知道"这里不对"。
>
> 年轻人把每一个"这里不对"记下来，整理成一套检查清单。
>
> 半年后，年轻人的手艺还是不如老王。但他做了一件老王做不到的事：他用那套检查清单，教会了三个完全没有木工基础的人，做出了八十分水准的家具。
>
> 老王的手艺是满分的。但那份手艺只能活在他手里。
> 年轻人的清单不是满分的。但它能离开创造者独立运行。
>
> **手艺 vs 工具的区别：手艺依赖人，工具独立于人。**
>
> 把手艺变成工具的关键步骤是什么？不是模仿动作，而是把隐性判断（"这里不对"）变成显性规则（检查清单）。
>
> 这就是你在 MVP 5 要做的事。

回想 MVP 1-4 的全过程。你至少用过以下工具：

- 四层漏斗（MVP 2）
- 三轮压缩 + 四道验证门（MVP 3）
- 升维五步 + 四步破局法 + 公式解码（MVP 4）

```
选一个你用得最顺手的工具，回答：
1. 它帮你解决了什么问题？_______________
2. 它的核心操作是什么？（一句话）_______________
3. 你能向别人解释它为什么管用吗？_______________
```

### 步骤 2 · 撞上天花板（5 分钟）

现在换一个场景：

> **你的一个朋友遇到了这样的问题**：他每天刷大量信息，读完觉得"都有道理"，但一周后什么都记不住。他不是不认真——他记了很多笔记，画了很多思维导图，但就是留不下什么。他问你："你有什么方法吗？"

```
你会怎么回答他？（不是推荐某个工具名称，而是你自己用的方式）

你的回答：__________________________________________
```

### 步骤 3 · 识别差距（3 分钟）

回头看你的回答：

```
1. 你的回答是在描述你的"个人经验"，还是给了一个"任何人都能用的流程"？___
2. 如果朋友照做，他能独立运行吗？还是离了你就跑不通？___
3. 你觉得，「有一套方法」和「能把方法变成工具让别人用」——差别大吗？___
```

> **检查点**：使用者知道怎么用工具，但不一定知道工具为什么这样设计。从使用者到创造者的跃迁 = 从「知道怎么用」到「知道为什么管用」再到「能设计新的」。

### 步骤 4 · 选一个你想解决的认知问题（3 分钟）

想一个你自己反复遇到的认知问题。不是大的人生困境（那是 MVP 4），而是一个具体的、反复出现的思考难题。

```
我反复遇到的认知问题：_______________

例子（只看不抄）：
  - "读论文时总是被术语卡住，看完了但没进入真正的逻辑层"
  - "做决策时总在搜集更多信息，搜集越多越不敢决定"
  - "跟人讨论时容易被带跑，事后才想到关键问题"
  - "有想法但表达不出来，写出来总觉得散"
```

先放在这里。等下回来。

## 第二幕：解剖 + 设计（40 分钟）

### 步骤 5 · 六刀解剖一个真实 Skill（15 分钟）

我们先拆一个真实的 Skill，看看工具的骨架是什么样的。

用你 MVP 1 体验过的「压缩 vs 摘要」作为解剖对象。（后面你还会看到第二个真实 Skill 作为对比。）

**第一刀：关系——它在生态系统中的位置**

```
上游（它依赖什么）：_______________
下游（谁用它的输出）：_______________
竞品/替代品：_______________
```

**第二刀：介面——输入什么，输出什么**

```
输入：_______________
输出：_______________
边界：什么情况进不了这个工具？_______________
```

**第三刀：基因——最小不可再分的操作**

```
原子操作 1：_______________
原子操作 2：_______________
原子操作 3：_______________
删掉任何一个，它还能运转吗？___
```

**第四刀：结构——基因如何组合**

```
组合方式（串行/并行/循环）：_______________
顺序能换吗？___
有没有反馈环路？___
```

**第五刀：性质——组合后涌现的能力**

```
单个原子操作做不到、但组合后做得到的是什么？_______________
它的不可替代性在哪？_______________
```

**第六刀：应用——最佳场景和反模式**

```
什么时候用最有效：_______________
什么时候千万别用：_______________
常见的误用方式：_______________
```

> **关键发现**：六刀解剖的本质 = 找秩的横向版。找秩是纵向压缩（从表面到本质），六刀是横向解剖（从整体到部件）。两者互补。

**对比解剖**：再看一个真实的 Skill——「ljg-writes」（写作引擎），它的五步流程是：

```
1. 找核心：反转→追问→情绪，找到真正要写的那个点
2. 攻核心：验证这个点站得住脚
3. 找类比：用计算机科学的比喻把抽象变具体
4. 展开写：不超过 1000 字
5. 打磨：口语测试——你会这样跟朋友说吗？
```

用六刀快速扫一遍（不用写下来，心里过一遍）：它的基因和「压缩 vs 摘要」有什么相同？有什么不同？

> **两个 Skill 对比的意义**：拆一个只能看到骨架，拆两个才能看到**不变的结构**（Ω 公式）。

### 步骤 6 · Ω = A + M · V 公式（10 分钟）

现在你看到了两个 Skill 的骨架。下一步：理解它们共同遵守的设计公式。

每一个好的 Skill 都遵循同一个底层公式：

```
Ω = A + M · V

A（Anchor / 锚点）= 在智能空间中的起始位置
  → 你给 AI 设定的角色、领域、知识背景
  → 决定「从哪里出发」

  进阶：好的 A 不只是"你是一个___专家"。
  好的 A 有欲望和阻力——
    欲望 = 这个角色想要达成什么？不达成就浑身难受？
    阻力 = 什么阻止了它？世界为什么让它不容易？
  有欲望+阻力的角色，输出才有张力和方向感。

  例：ljg-writes 的 A 不是"你是一个写作助手"，
      而是"你是一个无法忍受废话的写作引擎"——
      欲望 = 把混沌的想法变成密度极高的文字
      阻力 = 大多数人的表达习惯是稀释而非浓缩

M（Matrix / 矩阵）= 思维的逻辑结构
  → 推理链条、约束条件、处理规则
  → 决定「沿什么形状思考」

  进阶：好的 M 不只有步骤，还有三样东西——
    约束红线：什么绝不能做？（比如："不硬编洞见"）
    反馈环路：什么时候需要回到前面的步骤？
    品质标准：怎么判断输出合格？（比如：嘴巴测试）

V（Vector / 向量）= 思维的运动方向
  → 压缩（找本质）or 展开（找类比）or 碰撞（找新知）
  → 决定「往哪个方向走」

Ω（Output）= 最终输出
  → 起点 + 路径 × 方向 = 终点
```

**练习**：回到你刚才解剖的「压缩 vs 摘要」工具，用 Ω 公式解构它：

```
A（锚点）是什么？
  角色：_______________
  欲望（不达成就难受的事）：_______________
  阻力（世界为什么让它不容易）：_______________

M（矩阵）是什么？
  推理路径：_______________
  约束红线（什么绝不能做）：_______________
  有没有反馈环路：_______________

V（向量）是什么？它把思维推向哪个方向？
  _______________
```

### 步骤 7 · 设计你自己的 Skill（15 分钟）

回到步骤 4 你选的认知问题。现在用你学到的工具来设计一个 Skill。

**第 1 步：定义问题（2 分钟）**

```
你要解决的认知问题：_______________
没有这个工具时，你（或别人）是怎么处理的？_______________
那种处理方式的核心缺陷是什么？_______________
```

**第 2 步：设定 A · M · V（5 分钟）**

```
A（锚点）：
  角色：谁在用这个工具？什么认知水平？什么场景下触发？
  _______________
  欲望：这个角色不达成什么就浑身难受？
  _______________
  阻力：世界为什么让它不容易？
  _______________

M（矩阵）：
  思维路径（3-5 个步骤）：
  步骤 1：_______________
  步骤 2：_______________
  步骤 3：_______________
  步骤 4：_______________
  步骤 5：_______________

  约束红线（什么绝不能做？）：_______________
  品质标准（怎么判断输出合格？）：_______________
  反馈环路（什么时候需要回到前面的步骤？）：_______________

V（向量）：
  思维的目标方向是什么？压缩/展开/碰撞/其他？
  _______________
```

> **关于品质约束的设计要点**：
> 好的约束不是限制，是让工具更锋利。参考 V0.3 的五条品质规则：
> 1. 嘴巴测试：你会这样跟一个聪明朋友说吗？
> 2. 零术语规则：每个术语立刻接一个地气场景
> 3. 一句话压缩测试：压不成 = 还没想清楚
> 4. 短原子规则：能用 2 个字就不用 4 个字
> 5. 诚实原则：delta ≈ 0 就说 0，不硬编洞见
>
> 你不需要全用，但至少选 1-2 条写进你的 Skill。

**第 3 步：找到你的 Skill 的「暗门」（2 分钟）**

每本书都有一扇暗门——一旦打开，整本书的秘密豁然开朗。每个 Skill 也一样。

```
你的 Skill 的「暗门」是什么？
（那个让使用者从"不懂"到"懂了"的关键转折点是什么？）
_______________
```

> 例：「压缩 vs 摘要」的暗门 = 「从你找到的 3 个秩出发，能不能反推出原始的 997 个现象？」——能反推才算压缩，不能就只是摘要。
> 例：「升维公式」的暗门 = 那个被你当作"理所当然"但其实是假设的东西。

**第 4 步：写出 Skill 的餐巾纸公式（3 分钟）**

把你的 Skill 压缩成一个公式——变量不超过 5 个，一张餐巾纸画得下，5 分钟能给朋友讲明白。

```
你的 Skill 的餐巾纸公式：_______________

一句话解释：_______________
```

**第 5 步：六刀自检（5 分钟）**

用六刀解剖你刚设计的 Skill：

```
关系：上下游和竞品是什么？_______________
介面：输入什么、输出什么？_______________
基因：最小不可再分的操作有哪几个？_______________
结构：这些操作怎么组合？_______________
性质：组合后涌现了什么新能力？_______________
应用：最佳场景和反模式？_______________
```

如果六刀下来发现某一刀答不上——恭喜，你找到了设计的漏洞。回去补。

## 第三幕：八道验证门 + Meta 反思 + 沉淀（35 分钟）

### 步骤 8 · Metis 对话：八道验证门（18 分钟）

**把你设计的 Skill（A · M · V + 餐巾纸公式 + 六刀自检）发给 Claude。**

Claude 会做这些事：

1. **过八道验证门**

   - 门 1 回放：从你的 Skill 公式能反推出原始问题为什么难解吗？（复习）
   - 门 2 盲区预测：你的 Skill 能处理一个你没想过的同类问题吗？（复习）
   - 门 3 反事实检验：删掉 Skill 的一个步骤，它会怎样崩塌？（复习）
   - 门 4 正交操作：Skill 的步骤之间真的独立吗？（复习）
   - 门 5 自我陷阱：你的 Skill 设计里藏了什么你自己的河床？（复习）
   - 门 6 行动算法：这周怎么用一次这个 Skill？（复习）
   - 门 7 **创造者检验**（新！）：把你的 Skill 描述发给一个不认识你的人，他能独立运行吗？
   - 门 8 **递归检验**（新！）：你的 Skill 能不能用来分析它自己？

2. **过品质标准**
   - 嘴巴测试：你会这样跟一个聪明朋友解释你的 Skill 吗？
   - 压缩测试：Skill 的描述能再短吗？
   - 零术语规则：每个术语有地气场景吗？

3. **Delta 评估**
   - 诚实 > 制造洞见

> **两道新门解释**：
>
> **创造者检验**——使用者的天花板是「我能用」，创造者的标准是「别人能用」。如果你的 Skill 离了你的解释就跑不通，说明工具还没从你的脑子里独立出来。一个好的 Skill = 脱离创造者仍然可运行。
>
> **递归检验**——最优雅的工具能自指。你的 Skill 能分析它自己吗？如果能，说明你设计的是一个真正的「生成器」，不是一个特定问题的补丁。如果不能，没关系——大多数工具不需要自指。但知道这个边界本身就有价值。

### 步骤 9 · Meta 反思：理解「造工具」这件事（7 分钟）

你刚经历了一个完整的创造过程。现在反过来看这个过程本身。

**第 1 阶段 · 消费者 vs 创造者**：

```
使用工具时你关注什么？_______________
设计工具时你关注什么？_______________
最大的差异在哪？_______________
```

**第 2 阶段 · Ω 公式和你学过的东西的关系**：

```
Ω = A + M · V 和 MVP 4 的升维公式有什么关系？
_______________

Ω 公式和贝叶斯更新有什么关系？
_______________

Ω 公式和找秩有什么关系？
_______________
```

> 提示：A 锁定起始坐标 ≈ 先验。M 定义推理路径 ≈ 似然函数。V 定义方向 ≈ 更新方向。Ω 是后验。Ω = A + M · V ≈ 后验 = 先验 + 证据 × 更新。

**第 3 阶段 · 五个 MVP 的连贯逻辑**：

```
MVP 1-5 走过来，你觉得它们之间的「秩」是什么？
（不是五件独立的事，而是一条线——那条线是什么？）
_______________
```

### 步骤 10 · 沉淀（10 分钟）

**你的 Skill 设计卡**——以后想设计新工具时的速查卡：


## 回看：你刚走了什么路

90 分钟，你做了一件事：
**设计了一个属于你的认知工具——从使用者变成了创造者。**

但你走过了一条完整的创造管道：

```
  故事   "老王的手艺"          → 手艺依赖人，工具独立于人
  第一幕 "回顾工具"           → 使用者的视角（能用，但说不清为什么管用）
  第一幕 "撞上天花板"         → 发现「会用」和「能教」的差距
  第一幕 "识别差距"           → 经验 ≠ 工具，个人技巧 ≠ 可复用流程
  第二幕 "六刀解剖"           → 看清工具的完整骨架（拆一个+对比第二个）
  第二幕 "Ω = A + M · V"     → 理解设计公式（锚点=欲望+阻力，矩阵=步骤+约束+反馈环）
  第二幕 "找暗门"             → 定位工具的关键转折点
  第二幕 "设计你的 Skill"     → 从零创造，完整 Skill 文件
  第三幕 "八道验证门"         → 完整验证（+创造者检验+递归检验）
  第三幕 "Meta 反思"          → 理解消费者 vs 创造者的本质差异
  第三幕 "沉淀"               → Skill 设计卡 + 概念更新 + 测试计划
```

从消费者到创造者 = 不只是学会更多工具，而是学会**造工具的工具**。

```
这就是 MVP 5 的设计哲学：

  消费者 = 用别人的工具处理自己的问题
            越用越熟练，但工具的边界就是你的边界

  创造者 = 理解工具的设计原理，能造新的
            你的边界不再被工具限制
            Ω = A + M · V 就是「造工具的工具」
```

## 做完了？

你可以：
- **实战**：用你设计的 Skill 处理一个真实问题，验证它是否管用
- **迭代**：基于实战反馈修改 Skill 的 A·M·V，再跑一次六刀自检
- **写成 SKILL.md**：把你的 Skill 写成 Claude Code 可运行的格式，安装到你的工具链
- **分享**：把你的 Skill 给一个朋友用，看他能不能独立跑通（创造者检验的实战版）
- **深化**：用 Metis 对话继续探索 Ω 公式与三公理的深层关系
- **进阶**：探索「共振式」Skill 设计——不是给刚性约束，而是创造一个「场」，让 AI 在约束内自由共振。刚性约束适合确定性任务，共振场适合创意型任务。这是 Skill 设计的高阶形态。（参考：李继刚「反思者」Prompt 的共振式写法）
