## 这个课程的设计哲学

本课程用 V0.3 的方法论来教 V0.3。

不是"扫完7个模块"——那是摘要式教学。
而是"深入做1件事，其他模块在做的过程中自然涌现"——这是压缩式教学。

你不需要提前了解 V0.3 有哪些模块。做完了自然就知道了。

---

## 我们要用的概念

**压缩 ≠ 摘要**

为什么选它：这是 V0.3 最重要的认知升级。
不提前解释它。做一次就懂了。

## 第一幕：你平时是怎么学的（7 分钟）

### 步骤 1 · 阅读材料（2 分钟）

请阅读以下材料：

---

> A 医生记住了 3000 种疾病对应的症状和用药方案。患者来了，他在脑子里搜索匹配项——找到就开药，找不到就转院。有一天来了个古怪病例，症状组合在教科书上找不到对应。他翻遍 3000 个词条，无解。
>
> B 医生只抓住了人体五大调节系统（免疫、内分泌、神经、循环、代谢）之间的互动关系。同一个古怪病例到了她手上，她不搜索匹配项——她问："哪个系统失衡了？失衡方向是什么？"从五个系统的关系中，推导出可能的病因。
>
> 一个棋手背了 2000 个定式。对手走出定式外的一步，他慌了。另一个棋手只理解三个原则：控制中心、子力协调、弱点攻击。面对任何新局面，他都有方向。
>
> 一个厨师有 500 道菜谱。给他一种没见过的食材，他不知道怎么用。另一个厨师只懂两件事：热量如何改变质地、五种味道如何平衡。任何食材到手，他都能现编一道菜。

---

### 步骤 2 · 做一次正常总结（3 分钟）

用你平时学东西的方式，把这段材料提炼成 3-5 个要点：

```
1. ___
2. ___
3. ___
4. ___
5. ___
```

写完就好，不用追求完美。

### 步骤 3 · 用你的总结推导新场景（2 分钟）

现在**遮住原文**，只看你写的要点。回答这个问题：

**"一个人想学好写作，按照材料的逻辑，有两种截然不同的学法。分别是什么？具体学什么内容？预期效果如何？"**

```
你的回答：__________________________________________
```

> **检查点**：写的时候流畅吗？还是觉得要点不够用，需要回去看原文？
>
> 记住这个感觉。第二幕会用到。

## 第二幕：换一种方式处理同样的材料（7 分钟）

### 步骤 4 · 找生成器（4 分钟）

回到同一段材料。这次不总结要点。问自己一个问题：

**"这三组人的故事，背后是不是同一个东西在驱动？如果是，那个东西能写成一个公式或一句话吗？"**

提示（如果卡住了，按顺序看，看一条试一下再看下一条）：
1. 第一类人拥有的和第二类人拥有的，本质区别是什么？
2. 为什么一种知识遇到新情况就失效，另一种不会？
3. 两者的区别，能不能用一个数学隐喻来表达？

写出你的公式 / 一句话：

```
___________________________________________
```

没有参考答案。挣扎是内化的代价。写得粗糙没关系——你正在做的事叫「压缩」。

### 步骤 5 · 再次推导同一场景（3 分钟）

从你的公式出发，再答一次同样的问题：

**"一个人想学好写作，两种学法分别是什么？具体学什么？预期效果？"**

```
你的回答：__________________________________________
```

**跟第一幕的回答比一下。**
- 这次更流畅了吗？更具体了吗？
- 你能推导出"具体学什么内容"了吗？（比如：一种是收集写作模板，另一种是掌握节奏控制、信息密度、读者期待管理三个生成器...）
- 如果两次差距明显——你刚才亲身体验了「摘要」和「压缩」的区别

---

## 第三幕：Metis 对话 + 沉淀（6 分钟）

### 步骤 6 · Metis 对话（4 分钟）

**把你的公式/一句话发给 Claude**（在 Claude Code 对话中）。

Claude 会做三件事：

1. **过两道验证门**
   - 回放：从你的公式能反推出材料中三组人的差异吗？
   - 盲区预测：你的公式能预测材料没提到的新场景吗？
2. **过品质标准**
   - 嘴巴测试：你会这样跟一个聪明朋友解释吗？
   - 压缩测试：能再短一点吗？
3. **指出盲区**（如果有）
   - 如果你的理解已经到位，Claude 会说"delta ≈ 0，没有新东西可加"
   - 诚实 > 制造洞见

> **Metis 规则**：Claude 不是你的助手。你和 Claude 都服从真理。
> Claude 有义务说"你错了"，你有权利反驳。

### 步骤 7 · 沉淀（2 分钟）

用你手边任何工具记录：

| 项目 | 内容 |
|------|------|
| **概念** | 压缩 |
| **我的定义** | ___________ |
| **置信度** | ___（0.5=大概是这样 / 0.7=能解释能举例 / 0.9=能教别人） |
| **体验笔记** | 第一幕 vs 第二幕的推导体验，差异在哪？ |

> 已有 `concepts.org` 就直接更新。没有也没关系——MVP 2 会搭工具链。

## 回看：你刚走了什么路

20 分钟，你只做了一件事：
**把同一段材料，先用"总结要点"的方式处理，再用"找生成器"的方式处理。**

但你触碰了 V0.3 的核心组件——不是因为课程安排了它们，而是做的过程中自然需要它们：

```
  第一幕 "总结要点"         → 你体验了摘要式认知的天花板
  第二幕 "找生成器"          → 你体验了「压缩漏斗」的核心操作
  两次 "推导新场景"          → 你用了「验证门」中的两道（回放 + 盲区预测）
  第三幕 Metis 对话          → 你体验了「AI 协作升级」+「品质标准」
  最后的记录                 → 你体验了「概念表沉淀」
```

你没有刻意"学习"这些模块。你深入做了一件事，模块在做的过程中自然涌现。

```
这就是 V0.3 的设计哲学：

  摘要式课程 = 告诉你有 7 个模块，每个体验一遍
                997 个点 → 100 个要点
                看完就忘

  压缩式课程 = 深入做 1 件事，模块在做中自然涌现
                997 个点 → 3 个生成器
                做完就懂
```

**这个课程本身，就是「压缩 ≠ 摘要」的活教材。**

## 做完了？

你可以：
- **巩固**：换个概念（比如「秩」或「Epiplexity」），再跑一遍同样的三幕
- **深化**：在 Claude Code 中继续 Metis 对话，打磨你的餐巾纸公式
- **进阶**：进入 MVP 2，学习用 Skill 工具做更高效的压缩

## 后续 MVP 预告

```
MVP 1（本节）：一个概念的体感               ← 你在这里
               你得到：压缩 vs 摘要的亲身体验
               重点：感受差异

MVP 2（下一节）：一篇文章的完整消化
               你得到：Skill 实战（xray-article）+ Emacs 工具链搭建 + Denote 命名
               重点：从手动压缩到工具辅助压缩

MVP 3（再下一节）：一本书的三轮压缩
               你得到：xray-book + 完整四道验证门 + 收件箱管道
               重点：多轮迭代 + 验证闭环

MVP 4：一个困境的升维
               你得到：完整升维公式 + 四步破局法 + 公式解码
               重点：从认知到行动

MVP 5：设计你自己的 Skill
               你得到：Ω=A+M·V + Skill 设计模式 + 六刀解剖
               重点：从使用者到创造者
```

每个 MVP 都深入做一件事，V0.3 的模块在做中自然涌现。

---

*这个课程设计本身用了 V0.3 的方法：Epiplexity 原则——同一个系统，投入的算力越多，能提取的结构越多。MVP 1 是低算力体感扫描，MVP 5 是高算力深度开采。系统没变，变的是你。*
