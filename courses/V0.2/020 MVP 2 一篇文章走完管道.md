
> **时长**：30 分钟
> **前置**：完成 MVP 1（你已经体验过「压缩 ≠ 摘要」的区别）
> **你需要**：Claude Code（当前对话）+ 任何文本记录工具
> **一句话**：亲身体验「裸眼阅读」和「漏斗透视」的区别。把一篇文章走完整条消化管道。

**使用方式**：
- **自学**：阅读本文档，按步骤操作
- **引导模式**：在 Claude Code 中输入 `/mvp2`，跟着对话走（推荐）

## 这个课程的设计哲学

MVP 1 你学了「找生成器」。但那是在玩具材料上做的——三个工整的小故事。

真实世界的文章更长、更杂、有噪声。手动压缩碰到真文章，你会遇到新困难：
- 不知从哪层切入
- 被表面问题带偏
- 觉得"好像懂了"但说不清到底学到什么

MVP 2 的核心体验：同一篇文章，先「裸眼读」，再用「四层漏斗」重新透视。你会发现漏斗能抓到裸眼漏掉的层。

这不是教你一个方法论。做一次就知道了。

---

## 我们要用的概念

**管道：读 → 拆 → 压 → 撞 → 沉 → 更新**

为什么选它：这是从"偶尔顿悟"到"系统消化"的关键升级。
不提前解释它。走完了自然就懂了。

## 第一幕：你平时是怎么读文章的（8 分钟）

### 步骤 1 · 自然阅读（3 分钟）

请阅读以下文章：

---

> 很多人觉得好餐厅一定是大酒楼——菜品全、选择多、气派。但一个反直觉的现象是：全世界最顶级的餐厅，往往很小。
>
> 东京的"数寄屋桥次郎"只有 10 个座位。主厨小野二郎做了 60 年寿司，菜单只有一种：当天的 omakase。没有选择，没有装修，没有分店。但他被认为是世界上最好的寿司师傅。
>
> 大型连锁餐厅有 200 道菜，需要中央厨房、标准化流程、冷冻预制。每道菜都"还行"，但没有一道让你记住。
>
> 这不仅发生在餐饮业。Basecamp 只有 70 个员工，却做出了影响全球的产品。创始人说："如果你什么都想做，你就什么都做不好。少不是妥协，少是武器。"
>
> 生物学里也一样。免疫系统不试图记住所有病毒——那不可能。它只掌握几种核心识别模式，用这些模式推断新威胁。当免疫系统试图"记住一切"时，反而崩溃——那叫自身免疫病。
>
> 约束不是限制。约束是一种压力，迫使你放弃大多数选项，只保留真正重要的。被放弃的不是损失，而是噪声。留下来的，才是信号。

## 第二幕：四层漏斗透视同一篇文章（12 分钟）

### 步骤 4 · 第一层：找真问题（3 分钟）

回到同一篇文章。这次不自由阅读。

这篇文章标题是"为什么好餐厅都很小"。但作者**真正在回应的问题**，是这个吗？

如果他真正关心的只是餐厅大小，为什么要提到 Basecamp 和免疫系统？

**表面问题之下，作者真正想回答的是什么？**

```
真问题：__________________________________________
```

提示（卡住了再看）：
1. 去掉餐厅的例子，文章还成立吗？成立的话，核心不在餐厅。
2. 三个案例（餐厅、Basecamp、免疫系统）的**共同点**是什么？

从你找到的真问题出发，作者是怎么一步步论证的？

用 A → B → C → D 的形式画出来：

```

```

### 步骤 6 · 第三层：找生成器（3 分钟）

经过两层过滤，现在再找一次生成器：

**这篇文章的「秩」是什么？能压成一个公式或一句话吗？**

```
生成器（第二次）：___________________________________
```

**跟步骤 2 你的第一次尝试比一下。**
- 这次更精确了吗？
- 经过前两层过滤再找生成器，和裸眼直接找，感觉有什么不同？

### 步骤 7 · 第四层：认知碰撞（3 分钟）

这一层不是分析文章，是分析**你自己**。

看一眼你的概念表：

| 概念 | 你的定义 | 置信度 |
|------|----------|--------|
| 约束 | 下层约束产生上层自由 | 0.85 |
| 压缩 | （你在 MVP 1 的定义） | ~0.7 |
| 学习 | 改变了行为和判断的才叫学到 | 0.9 |

**这篇文章有没有改变你对其中任何一个的理解？**
- 哪个概念的置信度需要调整？
- 定义需要修改吗？怎么改？

```
碰撞记录：__________________________________________
```

> 如果没有碰撞，也是正常的。说明你的已有模型已经覆盖了这篇文章。

## 第三幕：管道沉淀 + Metis 对话（10 分钟）

### 步骤 8 · 给文件命名（2 分钟）

假设你要把这篇文章的消化结果保存成一个文件。文件名格式：

```
YYYYMMDDTHHMMSS--关键词__标签.org
```

关键词是你对文章本质的压缩。标签是分类。

试试看：

```
你的文件名：__________________________________________
```

> 命名即压缩。一个好文件名 = 一次成功的一句话压缩测试。

### 步骤 9 · Metis 对话（5 分钟）

**把你的最终版生成器发给 Claude**（在 Claude Code 对话中）。

Claude 会做这些事：

1. **过两道验证门**
   - 回放：从你的公式能反推出文章三个领域（餐饮/科技/生物）吗？
   - 盲区预测：你的公式能预测文章没提到的领域吗？
2. **过品质标准**
   - 嘴巴测试：你会这样跟一个聪明朋友说吗？
   - 压缩测试：能再短一点吗？
3. **检查文件名**
   - 准确吗？能搜到吗？够压缩吗？
4. **Delta 评估**
   - 诚实 > 制造洞见

> **Metis 规则**：Claude 不是你的助手。你和 Claude 都服从真理。

### 步骤 10 · 沉淀（1 分钟）

用你手边任何工具记录：

| 项目 | 内容 |
|------|------|
| **新概念或更新** | ___________ |
| **新的定义** | ___________ |
| **置信度变化** | ___（哪个概念变了？变了多少？为什么？） |

## 回看：你刚走了什么路

30 分钟，你做了一件事：
**消化一篇文章。**

但你走过了一条完整的管道——不是因为课程安排了它们，而是消化文章的过程中自然需要它们：

```
  第一幕 "自然阅读"          → 你体验了认知天花板（读 ≠ 消化）
  第二幕 "找真问题"           → 你体验了文章透视的核心操作
  第二幕 "画推理链"           → 你体验了论证骨架分析
  第二幕 "找生成器"           → 你复习了压缩漏斗（MVP 1）
  第二幕 "认知碰撞"           → 你体验了 baseline loading（概念表 = 认知基线）
  第三幕 "Denote 命名"        → 你体验了沉淀管道的入口（文件名 = 压缩测试）
  第三幕 "概念表更新"          → 你体验了概念表维护
  第三幕 "Metis 对话"         → 你体验了 AI 协作 + 品质标准
```

管道 = **读 → 拆 → 压 → 撞 → 沉 → 更新**

```
这就是 MVP 2 的设计哲学：

  没有管道的学习 = 读完就忘，偶尔顿悟
                    信息经过你，但没有留下痕迹

  有管道的学习   = 每篇文章走完同一条路
                    读 → 拆 → 压 → 撞 → 沉 → 更新
                    信息变成你的认知结构的一部分
```

**下次碰到任何文章，同样的管道再跑一遍。这不是"学习方法"，这是认知消化系统。**
