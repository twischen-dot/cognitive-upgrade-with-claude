## 这个课程的设计哲学

MVP 1 你学了「找生成器」。MVP 2 你走了一条消化管道。

但两次你都是一轮搞定。对短文章，一轮够用。面对更长更复杂的材料——一本书——一轮压缩会遇到新困难：
- 骨架抓了，但论证链没看透
- 生成器找了，但隐含假设没挖出来
- 碰撞做了，但跨域迁移没想到

MVP 3 的核心体验：同一本迷你书，先「单轮压缩」，再用「三轮迭代」重新深挖。你会发现每一轮聚焦不同层面，深度逐轮递增。

加上两道新的验证门（反事实 + 正交），和一条完整的收件箱管道。

做一次就知道了。

---

## 第一幕：你平时是怎么消化一本书的（10 分钟）

### 步骤 1 · 阅读迷你书（5 分钟）

请阅读以下迷你书（三章）：

---

> **为什么"不怕犯错"的人学得更快**
>
> **第一章 两种学生**
>
> 教钢琴的老师都知道一个怪现象。
>
> 有一类学生，练琴的时候小心翼翼。遇到一个不确定的音符，手指悬在键盘上方，迟迟不落下。他们宁可慢三倍，也不愿弹错一个音。
>
> 另一类学生正好相反。他们一头扎进去，噼里啪啦，错音不断。但奇怪的是，三个月后，第二类学生往往弹得比第一类好。
>
> 这不只发生在钢琴课上。学编程的人也分两种。一种人写一行代码就编译一次，确保每一步都没有 bug。另一种人先写出一大段"大概对"的代码，运行，看哪里报错，然后逐个修。后者上手更快。
>
> 学投篮也是如此。研究发现，练习期间命中率更低的那组球员——那些大量尝试不同角度和力度的人——在最终测试中的表现反而更好。
>
> 为什么犯更多错的人，最终学得更好？
>
> **第二章 错误的三张面孔**
>
> 答案藏在一个区分里：不是所有错误都一样。
>
> 第一种错误是噪声型错误。手滑了，注意力溜了，纯粹的随机失误。这种错误没有信息量。弹错了因为打了个哈欠，这告诉你什么？什么也没告诉你。
>
> 第二种错误是边界型错误。你试了一个你不确定行不行的做法，发现不行。这种错误极其宝贵——它告诉你"规则的边缘在哪里"。钢琴学生试着用一种新指法弹一段难的乐句，发现手指够不到。这个"够不到"就是一条边界。知道了边界在哪，你就知道了可以活动的空间有多大。
>
> 第三种错误是系统型错误。你有一个关于"事情应该怎么运作"的心理模型，但现实给了你一个完全出乎意料的结果。这说明你的模型本身有问题。这种错误的信息量最大——它不是在调整参数，而是在重写底层代码。
>
> 那些"不怕犯错"的学生，并不是在制造更多噪声。他们是在快速触碰边界，快速暴露系统错误。每一个有效错误，都是一次信号——"你的模型在这里失效了，请更新。"
>
> **第三章 学习系统的免疫力**
>
> 生物学里有一个完美的类比。
>
> 免疫系统不是靠"避免接触病毒"来保护你的。恰恰相反——它靠的是受控地暴露于威胁之中。疫苗就是故意引入的"弱化版错误"，让免疫系统在安全条件下练习识别和应对。
>
> 一个从小被养在无菌室里的孩子，免疫系统极其脆弱。一个在泥地里打过滚、吃过沙子的孩子，免疫系统反而强壮。
>
> 学习的逻辑一模一样。那些小心翼翼避免犯错的学生，就像生活在无菌室里——看起来干净，实际上脆弱。他们的模型没有经过真实错误的锤炼，遇到新情况就崩溃。
>
> 但这里有一个关键前提：暴露必须是受控的。如果免疫系统一下子面对太猛烈的病毒，不是变强，而是直接死亡。学习也一样——如果错误的后果太严重（比如学开飞机时把飞机撞了），那"不怕犯错"就不是勇敢，而是愚蠢。
>
> 真正高效的学习，需要两个条件同时满足：足够多的有效错误暴露，加上足够安全的犯错环境。
>
> 这就是为什么模拟器如此重要——飞行模拟器、手术模拟器、编程的沙盒环境。它们做的事情本质上都一样：在零风险的条件下，最大化你的错误产量。
>
> 当你理解了这一点，"学习"这件事就变得清晰了：学习不是信息的搬运，而是模型的迭代。错误不是学习的副产品，错误就是学习本身。没有错误的学习，就像没有阳光的光合作用——看起来在运转，实际上什么也没长出来。

## 第二幕：三轮迭代压缩同一本书（20 分钟）

### 步骤 4 · Round 1：骨架扫描（5 分钟）

回到同一本迷你书。这次不急着找生成器。先回答"这本书在说什么"：

**1. 作者真正想回答的问题是什么？**（不是标题里的问题——如果只关心"怕不怕犯错"，为什么要写第二章和第三章？）

```
真问题：__________________________________________
```

**2. 作者的回答是什么？**（一句话）

```
核心答案：__________________________________________
```

**3. 每章的核心论点是什么？**（每章不超过 10 个字）

```
第一章：_______________
第二章：_______________
第三章：_______________
```

提示（卡住了再看）：
1. 三个章节的深度在递进——第一章是现象，第二章是机制，第三章是条件+结论
2. 作者真正关心的不是"犯错"本身，而是犯错背后的某种学习机制

### 步骤 5 · Round 2：血肉解剖（7 分钟）

这一轮回答"凭什么这么说"：

**1. 推理链**：从问题到结论，作者怎么一步步论证的？

```
_____ → _____ → _____ → _____ → _____
```

**2. 关键证据**：最有说服力的 3 个证据/案例

```
1. _______________
2. _______________
3. _______________
```

**3. 隐含假设**：作者没说但必须成立的前提

```
隐含假设：__________________________________________
```

提示：第三章有一个关键的"但是"——那个前提如果不成立，整个论证会怎样？

**4. 边界条件**：什么情况下作者的结论会失效？

```
失效场景：__________________________________________
```

### 步骤 6 · Round 3：灵魂提取（5 分钟）

这一轮回答"还能怎么用"：

**1. 再找一次生成器**（经过两轮深挖后）：

```
生成器（第二次）：__________________________________________
```

**跟步骤 2 你的第一次比一下。**
- 这次更精确了吗？更有层次了吗？
- 经过两轮深挖再找生成器，和裸眼直接找，感觉有什么不同？

**2. 餐巾纸图**：如果只能画一张图给朋友看，画什么？

```
（文字描述即可）：__________________________________________
```

**3. 跨域映射**：这个洞见在其他领域叫什么？

```
投资领域：__________________________________________
育儿领域：__________________________________________
AI 训练领域：__________________________________________
```

**4. 认知碰撞**：

看一眼你的概念表：

| 概念 | 你的定义 | 置信度 |
|------|----------|--------|
| 学习 | 改变了行为和判断的才叫学到 | 0.9 |
| 约束 | 下层约束产生上层自由 | 0.85 |
| 压缩 | （你在 MVP 1 的定义） | ~0.7 |

**这本迷你书有没有改变你对其中任何一个的理解？**

```
碰撞记录：__________________________________________
```

> 步骤 7 · 对比
>
> 三轮走下来，和第一幕直接找生成器，感觉有什么不同？
> 如果差异明显——你刚体验了「单轮」和「三轮迭代」的区别。
> 如果差不多——这本迷你书不长，差异可能不明显。面对 300 页的书，多轮的价值会指数级放大。

## 第三幕：四道验证门 + 管道沉淀（15 分钟）

### 步骤 8 · Metis 对话：完整四道验证门（7 分钟）

**把你的最终版生成器发给 Claude**（在 Claude Code 对话中）。

Claude 会做这些事：

1. **过四道验证门**
   - 门 1 回放：从你的公式能反推出三章的核心吗？（复习）
   - 门 2 盲区预测：你的公式能预测书中没提到的领域吗？（复习）
   - 门 3 **反事实检验**（新！）：逐个"关掉"一个变量——那个残缺世界长什么样？现实中有真实案例吗？
   - 门 4 **正交操作**（新！）：你的变量之间真的独立吗？找到一个变了另一个没变的真实案例

2. **过品质标准**
   - 嘴巴测试：你会这样跟一个聪明朋友说吗？
   - 压缩测试：能再短一点吗？

3. **Delta 评估**
   - 诚实 > 制造洞见

> **两道新门解释**：
>
> **反事实检验**——你说学习需要 A 和 B？好，关掉 A，学习变成什么样？描绘那个残缺的世界。如果你能清晰描述，说明你对 A 的角色理解透了。说不清，理解还不够。
>
> **正交操作**——A 和 B 真的独立吗？找一个案例：A 变了 B 没变。再找一个：B 变了 A 没变。找得到，说明它们是真正独立的生成器。找不到，说明秩还能再降。

### 步骤 9 · 完整收件箱管道（5 分钟）

这次不只命名——走完整条路：

**第 1 步 · 命名**：

```
文件名：YYYYMMDDTHHMMSS--___________.____.org
```

**第 2 步 · 保存**：Claude 会把你的消化结果保存到 `~/Documents/notes/`。内容包含三轮压缩结果、生成器、验证门记录。

**第 3 步 · 更新概念表**：

| 项目 | 内容 |
|------|------|
| **新概念或更新** | ___________ |
| **新的定义** | ___________ |
| **置信度变化** | ___（哪个概念变了？变了多少？为什么？） |

> 完整管道 = 阅读 → 三轮压缩 → 四道验证 → 命名 → 保存 → 更新概念表
> 每一步都实际做了，不是只讲不练。

## 回看：你刚走了什么路

45 分钟，你做了一件事：
**消化一本迷你书。**

但你走过了一条更深的管道——不是因为课程安排了它们，而是消化一本书的过程中自然需要它们：

```
  第一幕 "单轮压缩"          → 你体验了一次性压缩的天花板
  第二幕 "骨架扫描"           → Round 1：这本书在说什么？
  第二幕 "血肉解剖"           → Round 2：凭什么这么说？（隐含假设 + 边界）
  第二幕 "灵魂提取"           → Round 3：还能怎么用？（跨域 + 碰撞）
  第三幕 "回放 + 盲区预测"    → 复习两道验证门
  第三幕 "反事实检验"         → 新：关掉一个变量，描绘残缺世界
  第三幕 "正交操作"           → 新：检验变量是否真的独立
  第三幕 "收件箱管道"         → 完整闭环：命名 → 保存 → 更新概念表
```

三轮压缩 = **骨架 → 血肉 → 灵魂**
每一轮聚焦不同层面，深度逐轮递增。

```
这就是 MVP 3 的设计哲学：

  单轮压缩 = 一遍抓要点，像快速浏览
              有时够用，但容易错过深层结构

  三轮压缩 = 每轮问不同的问题，层层剥开
              第一轮不迷路，第二轮不被忽悠，第三轮跟自己碰撞
              材料越复杂，多轮的价值越大
```

**这个课程本身，就是「迭代」的活教材：你不是在学一个方法论，你是在体验迭代。先做一轮，再做三轮，差异在做中自然涌现。**
